## Codebase Patterns
- Backend uses `uv` for dependency management; install with `uv sync --extra dev`.
- Integration/e2e tests live at `backend/tests/e2e/` (package with `__init__.py`); snapshot fixtures in `backend/tests/e2e/fixtures/`. Reuse the existing stub classes pattern (`_StubAnthropic`, `_StubReplicate`) rather than importing mocks — the pattern is already mypy-clean.
- `json.loads(...)` is typed as `Any` under strict mypy; wrap with `cast(dict[str, Any], ...)` when loading snapshot fixtures to avoid `no-any-return`.
- Backend quality gates: `uv run ruff check .`, `uv run mypy`, `uv run pytest` (run from `backend/`).
- mypy is configured with `strict = true` and `files = ["app", "tests"]`; every new function needs type annotations.
- FastAPI app lives at `backend/app/main.py` and is served via `uvicorn app.main:app --reload`.
- CORS is pinned to `http://localhost:5173` for the Vite frontend.
- Tests use `fastapi.testclient.TestClient`; place them in `backend/tests/` with `__init__.py`.
- HTTP endpoints live under `backend/app/routers/<name>.py` and are attached to `app` in `main.py` via `app.include_router(...)`. Use `APIRouter(prefix="/api", tags=[...])` for all feature endpoints.
- Use `Annotated[UploadFile, File(...)]` style for FastAPI dependencies — avoids ruff B008 while staying mypy-clean.
- Shared Pydantic schemas shared across routers live in `backend/app/schemas/` (e.g. `app/schemas/animation.py`). Router-local request models stay in the router file.
- Filesystem paths for derived assets live in `backend/app/storage.py` (`IMAGES_DIR`, etc.). Import the module (not the constant) so tests can `monkeypatch.setattr(storage, "IMAGES_DIR", tmp_path)`.
- `backend/storage/` is gitignored; create subdirs with `mkdir(parents=True, exist_ok=True)` on first write.
- Anthropic SDK calls need `cast(ToolParam, ...)` / `cast(ToolChoiceToolParam, ...)` / `list[MessageParam]` typing because the Messages.create overloads reject free-form dicts under strict mypy.
- For Anthropic-backed endpoints, expose a module-level `get_anthropic_client()` factory so tests can `monkeypatch.setattr(router_module, "get_anthropic_client", lambda: stub)`; keep the real function returning `anthropic.Anthropic()` (reads `ANTHROPIC_API_KEY`).
- For Replicate-backed endpoints, follow the same factory pattern: `get_replicate_client()` returns `ReplicateClient()` (imported from `replicate.client`, NOT `replicate.Client` — the top-level re-export is flagged by mypy strict `attr-defined`).
- Derived layer assets are served via a `StaticFiles` mount in `app/main.py` (e.g. `/storage/layers`). Ensure the target directory exists (`mkdir(parents=True, exist_ok=True)`) before calling `app.mount`, otherwise Starlette raises at import time.
- Replicate's `Client.run(...)` return shape is `Any` in practice (bytes, URL str, file-like with `.read()`, or a 1-element list). Keep a small `_coerce_*` helper per endpoint to normalize before doing work.
- Frontend lives in `frontend/` (Vite 8 + Vue 3 + TS + Tailwind v4 via `@tailwindcss/vite`). Run commands from `frontend/`: `npm install`, `npm run dev` (port 5173, strictPort), `npm run build` (vue-tsc typecheck + vite build).
- `tsconfig.app.json` sets `erasableSyntaxOnly: true` — this rejects TypeScript parameter properties (`constructor(public foo: T)`) and enums. Declare fields explicitly and assign in the constructor body instead.
- API base URL lives in `frontend/src/lib/api.ts` (`API_BASE_URL`, defaulting to `http://localhost:8000`, overridable via `VITE_API_BASE_URL`). All HTTP calls should go through the `api` helpers or typed wrappers in that file.
- Routes are registered in `frontend/src/router/index.ts`; add views to `frontend/src/views/` and export the router via `export const router`. `App.vue` renders `<RouterView />` only.
- For upload progress, use `XMLHttpRequest` with `xhr.upload.onprogress` — `fetch()` cannot report upload progress in the browser. Keep the XHR helper in `frontend/src/lib/api.ts` and expose `{ onProgress, signal }` options.
- Client-side validation must mirror backend rules (PNG/JPEG/WebP, ≤10MB) — surface errors via `ApiError.body.detail` when the server rejects, else a generic network-error fallback.
- Pipeline orchestration pattern: represent each step as `{key, label, status, error}`; when retrying a step, reset it and all downstream steps to `pending` before re-running, so a failure mid-pipeline never leaves stale success state.
- Typed API wrappers for each endpoint live alongside `uploadImage` in `frontend/src/lib/api.ts`. `postJson` takes `JsonValue`, so richer TS types (e.g. `Element[]`) need `as unknown as JsonValue` at the call site — keep the cast at the wrapper boundary, not inside UI code.
- Server layer URLs are root-relative (`/storage/layers/...`); prefix with `API_BASE_URL` in the frontend so they resolve to the backend origin, not the Vite dev server.
- Segmented layer PNGs and `background.png` are all written at the original image's full dimensions (Pillow `putalpha` + LANCZOS resize), so frontend layer stacking just needs `absolute inset-0 w-full h-full object-contain` with a container sized by aspectRatio — no per-layer offsets or scaling needed.
- Reusable UI pieces live in `frontend/src/components/`; views in `frontend/src/views/` consume them. Use a default `<slot>` to let views inject overlays (e.g. bbox selection SVG) over layered content.
- Frontend unit tests use Vitest (`npm run test`). Config lives in `frontend/vitest.config.ts`; include pattern is `src/**/*.{test,spec}.ts`. Default environment is `node` (happy-dom is installed for future Vue component tests).
- GSAP's global types expose `GSAPTween`, `GSAPTimeline`, etc. as top-level aliases — use those in tests instead of `import type gsap from 'gsap'`, because `erasableSyntaxOnly`/`noUnusedLocals` reports type-only imports as unused.
- GSAP timelines accept any object as a tween target (not just DOM nodes), so animator unit tests can pass `{}` as the layer ref and assert `tween.vars.x`, `timeline.duration()`, `subTimeline.repeat()` without a DOM.
- Child components that wrap animated DOM (e.g. `LayeredCanvas`) expose layer handles via `defineExpose({ getLayerRefs })` — callers access them with `ref<InstanceType<typeof LayeredCanvas> | null>` + `canvasRef.value?.getLayerRefs()`. Keeps the DOM `<img>`s encapsulated while letting views feed them to `buildTimeline`.
- When rebuilding a GSAP timeline (user re-runs plan), dispose the previous one with `timeline.kill()` AND reset inline `style.transform` / `style.opacity` on each layer before building the new one — otherwise GSAP's residual transforms bleed into the next animation's initial state.
- `import type { AnimationDSL } from './animator'` inside `api.ts` keeps backend response typing in sync with the animator's DSL without duplicating interfaces (single source of truth). Avoid circular imports by keeping animator.ts free of api.ts imports.
- For npm packages that ship no types (e.g. `gifenc`), add a minimal `frontend/src/types/<pkg>.d.ts` ambient module declaration — `tsconfig.app.json` already globs `src/**/*.ts`, so no include changes are needed.
- GSAP's `gsap.getProperty(target, prop)` reads the current (animated) value at whatever time the timeline is paused at. Seek via `timeline.pause(t)` then read x/y/rotation/scale/opacity to drive offscreen canvas rendering of the same frame — no DOM-to-canvas libraries required.
- GSAP's default transform-origin is 50% 50%, so the canvas equivalent of a GSAP tween is `translate(w/2 + x, h/2 + y)` → `rotate` → `scale` → `drawImage(img, -w/2, -h/2, w, h)`. Wrap each layer draw in save/restore and use `ctx.globalAlpha` for opacity.
- GSAP x/y are in CSS-pixel space (what was set); the offscreen canvas runs at intrinsic image dims. Scale by `intrinsic/rendered` when translating so motion amplitude matches what the user saw in the browser.
- Fresh `new Image()` objects with `img.crossOrigin = 'anonymous'` let canvas read pixel data without tainting — FastAPI's top-level CORSMiddleware already wraps the `/storage/layers` StaticFiles mount, so no backend change is needed to support `canvas.getImageData` after drawing the layer PNGs.
